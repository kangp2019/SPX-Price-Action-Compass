# SPX Price Action Compass (SPX 프라이스 액션 나침반)

---

`SPX Price Action Compass`는 S&P 500 지수(SPX)에 특화된 고성능 인터랙티브 프라이스 액션(Price Action, 가격 행동) 분석 및 복기(백테스팅) 시스템입니다. 다중 시간프레임 차트, 일봉 기준 당일 5분봉 정밀 드릴다운, 프라이스 액션 패턴 자동 감지 및 지지/저항(S&R) 구간 분석을 통합 제공하여 트레이더가 시장 구조를 완벽하게 학습하고 이해할 수 있도록 지원합니다.

본 프로젝트는 완전히 오픈소스로 운영되며, 광고나 타사 추적 스크립트 없이 오직 기술적 분석과 최적화된 사용자 경험에만 집중하는 트레이더 및 연구자들을 위해 개발되었습니다.

## 🌟 주요 특징

- **📈 증권사 급 다중 시간프레임 차트 (Multi-Timeframe Charts)**
  - **1분봉 (1m)**, **5분봉 (5m)**, **15분봉 (15m)**, **4시간봉 (4h)**, **일봉 (1d)** 지원.
  - 백엔드가 실제 거래소 운영 시간의 데이터를 취합하여 끊김 없는 신뢰성 높은 대표 캔들 데이터를 생성합니다.

- **⚡ 빠른 로컬 캐싱 및 매일 자동 증분 업데이트 (Fast Local Caching)**
  - 효율적인 로컬 듀얼 JSON 파일 캐싱 메커니즘을 내장하고 있어 모든 과거 데이터가 서버 측에 안정적으로 저장됩니다.
  - 뉴욕 시간 매일 16:18(시장 마감 직후)에 **매일 데이터 증분 업데이트**가 자동으로 트리거됩니다. 불필요한 외부 API 요청을 최소화하여 단 1초 만에 모든 차트가 실시간으로 로드됩니다.

- **🔍 일봉에서 당일 5분봉으로 1-클릭 드릴다운 (Intraday 5m Drilldown)**
  - 일봉(1d) 차트 모드에서 과거 특정 일봉 캔들을 클릭하면, 해당 거래일의 **5분봉 초고해상도 인트라데이 추세 차트**를 백엔드에서 실시간 호출하여 화면에 시각화합니다.
  - 드릴다운 화면에서도 시가/고가/저가/종가 데이터 및 고정밀 인트라데이 요약 통계를 모두 제공하여 상세한 복기가 가능합니다.

- **🎯 전문가용 십자선 마우스 트래커 및 축 스케일 (Professional Crosshair)**
  - 전문 증권사 트레이딩 시스템과 같은 마우스 포인터 중심의 점선 십자선과 실시간 스케일 강조를 탑재했습니다.
  - 왼쪽 축(가격)과 하단 축(정확한 타임스탬프)이 마우스 위치에 반응하여 실시간으로 고대비 하이라이트됩니다.

- **🏷️ 자동 프라이스 액션 라벨링 알고리즘 (Auto Price Action Labeling)**
  - 기하학적 정밀 수학 모델을 활용하여 K-라인(캔들스틱) 및 차트 패턴의 프라이스 액션을 실시간으로 감지합니다 (서버 외부 AI API를 호출하지 않는 100% 오프라인 초고속 알고리즘).
  - 지지/저항 구조 및 스윙 극점: **HH** (Higher High), **LH** (Lower High), **HL** (Higher Low), **LL** (Lower Low) 자동 표시.
  - 과거 주요 거래 밀집도를 분석하여 지지 및 저항 밴드(S&R Zones)를 실시간으로 탐색 및 시각화.

## 🔬 프라이스 액션 감지 알고리즘 작동 원리

이 알고리즘은 **Al Brooks** 및 **Bob Volman**의 고전적인 프라이스 액션 이론을 기반으로, 성능 지연이 없는 결정론적 분석 알고리즘으로 설계되었습니다 (핵심 로직 파일: `/src/utils/patternDetector.ts`):

### 1. 국소 스윙 극점 감지 (Swing Points Detection)
* **원리**: 현재 캔들의 이전과 이후에 이동 평균 윈도우(Lookback/Lookforward)를 설정하고 국소 가격 한계치를 구합니다.
* **기준**: 특정 캔들의 `high`가 해당 캔들의 좌우 $N$개 캔들 내 모든 `high`보다 높은 경우 **Swing High**로 라벨링됩니다. 마찬가지로 `low`가 주변 좌우 $N$개 캔들의 `low`보다 엄격히 낮으면 **Swing Low**로 분류됩니다.

### 2. 시장 구조 및 트렌드 정의 (Market Structure)
* **HH (Higher High)**: 현재 고점이 이전 스윙 고점보다 높음.
* **LH (Lower High)**: 현재 고점이 이전 스윙 고점보다 낮음.
* **HL (Higher Low)**: 현재 저점이 이전 스윙 저점보다 높음.
* **LL (Lower Low)**: 현재 저점이 이전 스윙 저점보다 낮음.
* **추세 분석**: 최근 스윙 고/저점 배열과 캔들의 평균 기울기를 복합 가중하여 현재 추세 상태(상승, 하락, 또는 박스권) 및 강도를 실시간으로 계산해 냅니다.

### 3. 동적 지지 및 저항 대역 클러스터링 (S&R Zones Clustering)
* **원리**: 모든 과거 스윙 고점과 저점의 가격 좌표를 1차원 수직축 상에 투영합니다.
* **클러스터링**: 1차원 병합형 계층적 클러스터링(Agglomerative Clustering) 알고리즘을 사용합니다. 다수의 스윙 극점 가격들이 아주 미세한 오차 범위(예: $0.12\% - 0.15\%$) 내에 몰려있으면, 하나의 유효 가격대(클러스터)로 묶입니다.
* **유형 정의 및 가중**:
  - **저항선 (Resistance)**: 주로 스윙 고점들이 클러스터링된 대역.
  - **지지선 (Support)**: 주로 스윙 저점들이 클러스터링된 대역.
  - **매물대 전환 (S/R Flip)**: 고점과 저점이 혼재된 대역 (매우 강력한 지지/저항 역할).
  - **최적화**: 화면이 복잡해지는 것을 막기 위해 테스트 빈도가 2회 미만인 구간은 배제하고, 가장 많은 돌파 시도가 일어났던 상위 8개 구간만 정제하여 사용자 화면에 그립니다.

### 4. 캔들스틱 유형 감지 (Candlestick Patterns)
캔들의 **실체 크기, 꼬리 길이 비율, 그리고 직전 평균 변동성(ATR 유사 값)과의 관계**를 정의하는 명확한 기하학적 조건문으로 평가합니다:
* **도지 (Doji)**: 몸통 크기가 전체 변동 폭에 비해 극히 작고(`body / range < 8%`) 일반적인 가격 안정화 상태에서 발생하며, 힘의 균형 상태를 의미합니다.
* **핀바 (Pin Bar)**:
  - **상승형 핀바 (망치형)**: 아래꼬리가 몸통 대비 길고(`lowerShadow > range * 60%`), 위꼬리가 짧으며(`upperShadow < 15%`), 몸통 크기가 제한적(`bodyRatio < 30%`)인 형태. 가격 하락을 즉시 방어하는 강한 매수세를 증명합니다.
  - **하락형 핀바 (유성형)**: 위꼬리가 몸통 대비 길고(`upperShadow > range * 60%`), 아래꼬리가 짧으며(`lowerShadow < 15%`), 몸통 크기가 제한적(`bodyRatio < 30%`)인 형태. 고점 매물을 뚫지 못하고 강하게 밀린 매도세를 의미합니다.
* **인사이드바 (Inside Bar)**: 현재 캔들의 고점과 저점이 바로 이전 캔들의 가격 범위 내에 완전히 갇힌 구조. 변동성의 수축 상태로 큰 추세 돌파의 신호로 해석됩니다.
* **장대 캔들 (Engulfing)**: 현재 캔들의 실체(몸통)가 이전 캔들의 전체 실체를 완전히 역방향으로 덮으면서 총 변동 폭이 확대된 형태.
* **샛별/석별형 (Morning/Evening Star)**: 전형적인 3캔들 추세 전환 패턴. 첫 번째 장대 캔들, 두 번째 갭을 둔 미세 캔들(스타), 세 번째 역방향 장대 캔들의 깊은 관통력을 종합 분석해 감지합니다.

### 5. 차트 패턴 감지 (Chart Patterns)
* **이중 천정 (Double Top) / 이중 바닥 (Double Bottom)**: 인접한 두 스윙 고점 또는 저점의 가격 편차가 극도로 적고, 두 지점 사이 적당한 시간적 간격($8 - 40$ 캔들)이 발생해 미세 횡보 노이즈를 걸러낸 후 판단합니다.
* **헤드앤숄더 (Head & Shoulders / 삼존)**: 연속된 3개의 스윙 고점 중 중앙의 고점(헤드)이 양측 고점(좌/우 어깨)보다 눈에 띄게 높고 어깨 가격의 수평 대칭이 비교적 고르게 일치할 때 표시합니다.
* **삼각형 패턴 (Triangles)**: 3개의 고점이 점차 낮아지고 3개의 저점이 점차 높아지면 대칭 삼각형(Symmetrical Triangle)으로 판단하며, 단방향이 수평을 유지하면 상승/하락 삼각형으로 분류합니다.

## 🤝 기여와 최적화는 언제나 환영합니다!

구현된 알고리즘은 일반적인 가격 순응 시장에서 높은 신뢰도를 보이지만, 실제 금융 시장의 파동은 무수히 다양합니다. 고정 매개변수(꼬리 비율, 클러스터 수차 등)는 시장 주기나 추세 강도에 따라 개선될 여지가 있습니다.

**알고리즘 미세조정, 튜닝, 새로운 패턴 확장 등에 대한 기여를 환영합니다:**
- **클러스터링 알고리즘 개선**: 1차원 DBSCAN이나 밀도 기반 그리드 클러스터링을 적용해 지지/저항 밴드의 타겟 편차를 정교화하는 작업.
- **가변 임계치 도입**: ATR을 참조 지표로 삼아 핀바나 인사이드바 감지 비율을 시장 변동폭에 동적으로 추종하도록 조절하는 작업.
- **새로운 프라이스 액션 컨셉 지원**: FVG(Fair Value Gap) 및 OB(Order Block) 분석 로직 탑재.

알고리즘 상의 라벨 오차나 더 정교한 수학 모델링에 대해 의견이 있으시다면 언제든 Pull Request 및 이슈 제안을 남겨주세요.

## 🔄 지속적인 통합 및 배포 (CI/CD)

본 프로젝트는 **GitHub**, **Google Cloud Build**, **Google Cloud Run**을 유기적으로 연동하여 **100% 자동화된 웹앱 배포 파이프라인**을 제공합니다.

- **⚡ 연속적인 빌드 트리거 (Continuous Integration)**:
  GitHub 레포지토리의 `main` 브랜치로 새로운 푸시 및 머지가 감지되면, Google Cloud Build 웹훅이 이를 인지하고 빌드 파이프라인을 자동 호출합니다.
- **🐳 다단계 컨테이너 빌드 (Dockerized Build)**:
  - 프로젝트 루트의 `Dockerfile` 설정을 바탕으로, 안전하고 가벼운 `node:22-alpine` 환경에서 컴파일됩니다.
  - **Builder 단계**: 의존 모듈을 설치하고 빌드합니다 (`npm run build`). Express 서버(TypeScript) 소스를 단 하나의 초고속 로딩 CommonJS 파일(`dist/server.cjs`)로 패킹합니다.
  - **Runner 단계**: 생성된 결과물(`dist/`) 및 로컬 디렉토리(`data/`)만 이식하여 빌드 시점의 불필요한 노이즈와 도구들을 완전히 배제하고 컨테이너 이미지를 컴파일하여 빠른 콜드 스타트를 유지합니다.
- **🚀 무중단 무손실 배포 (Continuous Deployment)**:
  도커 이미지는 **Google Artifact Registry**로 안전하게 전송되어 수 초 만에 **Google Cloud Run** 서버로 업데이트됩니다. 무중단 롤링 배포를 준수합니다.

  - **React 18** & **TypeScript** — 탄탄한 컴포넌트 확장성과 정적 타입 설계 보장.
  - **Tailwind CSS** — 개발자 감성의 다크 슬레이트 테마(Dark Slate Theme) 고대비 컴팩트 레이아웃.
  - **자체 개발 SVG 캔버스 엔진** — 캔버스 라이브러리 용량 낭비 없이 브라우저 상에서 가볍고 역동적인 SVG 그래픽 렌더링을 구현하여 매끄러운 드래그(Pan) 및 줌(Zoom) 기능 제공.

- **백엔드 (Backend):**
  - **Node.js** & **Express** — 다중 시간대 캔들 데이터 배포 및 로컬 데이터 동기화 유지.
  - **Yahoo Finance API Proxy** — 안전한 보안 게이트웨이 역할 수행 및 4시간 등 특수 시간대 캔들 가공 서빙.
  - **크론 기반 스케줄러** — 오프라인 데이터 수집 자동화 일정 관리.

## 🚀 빠른 시작 가이드 (Getting Started)

### 1. 요구 사항
로컬 컴퓨터에 **Node.js (v18 이상)**가 올바르게 설치되어 있는지 확인하십시오.

### 2. 의존 모듈 설치
프로젝트 최상위 디렉토리에서 다음 명령을 실행합니다:
```bash
npm install
```

### 3. 개발 서버 실행
```bash
npm run dev
```
컴파일이 완료되면 브라우저에서 `http://localhost:3000`으로 접속하여 즉시 사용할 수 있습니다.

### 4. 프로덕션 빌드
```bash
npm run build
```
경량화 완료된 프론트엔드 번들과 컴파일된 서버 번들이 `dist/` 폴더 내부로 최적화 출력됩니다.
