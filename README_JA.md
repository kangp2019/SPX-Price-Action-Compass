---

`SPX Price Action Compass` は、S&P 500指数（SPX）に特化した、インタラクティブ性の高いプライスアクション（値動き）詳細分析・復盤（バックテスト）システムです。複数時間足チャート、日足からの5分足詳細ドリルダウン、自動プライスアクションパターン認識、および動的サポート・レジスタンス（S&R）検出機能を統合し、トレーダーが市場構造を迅速にマスターできるよう支援します。

本プロジェクトは完全にオープンソースであり、広告や不要なトラッキングコードは一切含まれていません。技術分析（テクニカル分析）とスムーズな操作性を重視するトレーダーや研究者のために設計されています。

---

## 🌟 主な機能

- **📈 複数時間足ブローカー級チャート (Multi-Timeframe Charts)**
  - **1分足 (1m)**、**5分足 (5m)**、**15分足 (15m)**、**4時間足 (4h)**、および **日足 (1d)** に対応。
  - バックエンドが自動的に指数取引時間の実際のデータを処理し、一貫性のあるローソク足チャートを生成します。

- **⚡ 毎日の自動増量更新と高速ローカルキャッシュ (Fast Local Caching)**
  - インテリジェントなローカルJSONファイル・ダブルキャッシュ機構を備え、すべての時間足のデータがサーバー側でオフライン保存されます。
  - ニューヨーク時間で毎日16:18（市場クローズ直後）に、**毎日のデータ増量更新**が自動的にトリガーされます。外部APIへの不要なリクエストを排除し、チャートの読み込みは一瞬で完了します。

- **🔍 日足から日内5分足への一クリック下钻 (Intraday 5m Drilldown)**
  - 日足（1d）チャートモードで、任意の過去のローソク足をクリックすると、バックエンドから**該当日の5分足超高解像度チャート**が動的に取得され、即座に表示されます。
  - ドリルダウンされたチャートでも、日内の始値・終値・高値・安値を含む詳細な統計が提供され、より詳細な復盤が可能です。

- **🎯 プロフェッショナルな十字カーソルと軸ラベル (Professional Crosshair)**
  - 大手ブローカー向けツールに匹敵する、マウス追従型の点線十字カーソルを搭載。
  - 左軸（価格）と下軸（正確なタイムスタンプ）が、マウスカーソルの位置に合わせてリアルタイムで高亮表示されます。

- **🏷️ 自動プライスアクション判定アルゴリズム (Auto Price Action Labeling)**
  - 高度な幾何学的数学モデルを使用し、K線（ローソク足）やチャートパターンのプライスアクションをリアルタイムでオフライン検出（外部AIに依存せず、ローカルで超高速に動作）。
  - スイングハイ・スイングローなどの市場構造極点：**HH** (Higher High), **LH** (Lower High), **HL** (Higher Low), **LL** (Lower Low) を自動ラベリング。
  - 過去の値動きの密度に基づき、主要なサポート＆レジスタンスゾーン（S&R Zones）を動的に描画。

## 📱 製品モジュール

システムには、学習から実践までの完全なサイクルを提供する2つの主要モジュールが含まれています：

### 1. 📈 プライスアクション学習・復盤 (Price Action Review)
* **機能紹介**: 過去の複数時間足チャートの復盤と学習のために設計されています。チャート上に波段極点（HH/LH/HL/LL）、主要なサポート・レジスタンス帯、および各種ローソク足パターンを純ベクトルSVGで可視化。右側の監視パネルから一クリックで特定のパターンに移動、ズーム、ドリルダウンが可能です。
* **インターフェースプレビュー**:
  ![プライスアクション学習・復盤](./pa-review.png)

### 2. ⚡ リアルタイム模擬チャレンジ (Real-time Challenge)
* **機能紹介**: 実戦向けのトレード判断能力と筋肉反射を鍛えるサンドボックス。システムは過去の膨大なデータからランダムに特定のSPX相場セグメントを切り出し、それ以降の値動きを隠した状態で提示します。ユーザーはこれまでの知識（プライスアクション、サポート・レジスタンス、出来高、モメンタム）を頼りに、買い（Long）、売り（Short）、または見送り（Skip）を判断します。判断後、未来のローソク足が再現され、仮想口座残高や勝率がリアルタイムで更新されます。
* **インターフェースプレビュー**:

## 🔬 プライスアクション検出アルゴリズムの仕組み

本アルゴリズムは、**Al Brooks** 氏と **Bob Volman** 氏の古典的なプライスアクション理論をベースにしており、確定性・低遅延のアルゴリズムを実装しています（コード：`/src/utils/patternDetector.ts`）：

### 1. 局所スイング極点の検出 (Swing Points Detection)
* **原理**: 現在のローソク足の前後にスライドウィンドウ（Lookback/Lookforward）を設定し、局所的な高値と安値を判定します。
* **条件**: あるローソク足の `high` が、その左右それぞれ $N$ 本のローソク足の `high` よりも高い場合、**Swing High** とマークされます。同様に、`low` が左右それぞれ $N$ 本の `low` よりも低い場合、**Swing Low** とマークされます。

### 2. 市場構造（トレンド構造）ラベリング (Market Structure)
* **HH (Higher High)**: 現在のスイング高値が前のスイング高値より高い。
* **LH (Lower High)**: 現在のスイング高値が前のスイング高値より低い。
* **HL (Higher Low)**: 現在のスイング安値が前のスイング安値より高い。
* **LL (Lower Low)**: 現在のスイング安値が前のスイング安値より低い。
* **トレンド判定**: 直近のスイング極点の発生パターンとローソク足の平均傾きから、現在の全体的な市場状態（上昇、下落、またはレンジ）を動的に出力します。

### 3. サポート＆レジスタンス帯の動的クラスタリング (S&R Zones Clustering)
* **原理**: 過去のスイング高値と安値の価格を一次元数軸上にマッピング。
* **クラスタリング**: 一次元凝集型階層クラスタリング（Agglomerative Clustering）アルゴリズムを採用。複数の極値価格が非常に小さな許容差（例：$0.12\% - 0.15\%$）以内に集中している場合、一つの価格帯（クラスター）にグループ化されます。
* **判定とフィルタリング**:
  - **レジスタンス (Resistance)**: 主にスイング高値が集まったクラスター。
  - **サポート (Support)**: 主にスイング安値が集まったクラスター。
  - **互換位 (S/R Flip)**: スイング高値と安値の両方が混在するクラスター（古典的なサポート・レジスタンス転換）。
  - **最適化**: チャートの乱雑さを避けるため、テスト回数が2回未満のゾーンはノイズとして排除し、最も強固な上位8つのサポート・レジスタンス帯のみを描画します。

### 4. 主要ローソク足パターンの認識 (Candlestick Patterns)
ローソク足の**実体の大きさ、上下のヒゲの比率、および過去の平均ボラティリティ（ATR近似）との関係**に基づき、厳格な幾何学的条件で分類します：
* **Doji (十字星)**: 実体がローソク足全体の範囲に対して極めて小さい（`body / range < 8%`）もので、多空が完全に均衡している状態。
* **Pin Bar (ピンバー / ハンマー / シューティングスター)**:
  - **強気（買い）ピンバー**: 下ヒゲが極めて長い（`lowerShadow > range * 60%`）、上ヒゲが短い（`upperShadow < 15%`）、実体が小さい（`bodyRatio < 30%`）。安値圏での強い買い圧力を示します。
  - **弱気（売り）ピンバー**: 上ヒゲが極めて長い（`upperShadow > range * 60%`）、下ヒゲが短い（`lowerShadow < 15%`）、実体が小さい（`bodyRatio < 30%`）。高値圏での強い売り圧力を示します。
* **Inside Bar (インサイドバー / はらみ足)**: 現在のローソク足の最高値と最安値が、一本前のローソク足の範囲内に完全に収まっている状態。ボラティリティの収縮を示し、大きなブレイクアウトの前兆となることが多いです。
* **Engulfing (エンガルフィング / 包み足)**: 現在のローソク足の実体が、一本前のローソク足の実体を完全に逆方向に包み込み、ボラティリティが拡大した状態。
* **Morning/Evening Star (明けの明星 / 宵の明星)**: クラシックな3本ローソク足パターン。1本目の大陽線/大陰線、2本目の極小実体（スター）、3本目の深い逆方向への貫通力を総合的に判定。

### 5. チャートパターンの認識 (Chart Patterns)
* **ダブルトップ (Double Top) / ダブルボトム (Double Bottom)**: 連続する2つのスイング極値高値/安値を検出し、それらの価格差が非常に小さく、かつ時間軸上で適切な距離（例：$8 - 40$ 本のローソク足）が離れている場合に判定。微小なもみ合いノイズを排除します。
* **三尊天井 (Head & Shoulders)**: 連続する3つのスイング高値を検出。中央の極値（ヘッド）が左右（左肩・右肩）よりも明確に高く、かつ肩の高さがほぼ対称的である場合にマーク。
* **三角保ち合い (Triangles)**: 連続する3つの局所高値が右下がり、かつ3つの局所安値が右上がりの傾きを示す場合に対称三角形（Symmetrical Triangle）と判定。一方が水平な場合は、上昇/下降三角形と判定されます。

## 🤝 コントリビューション・最適化歓迎！

現在の確定性アルゴリズムは通常の環境では高い精度を発揮しますが、金融市場は複雑で、固定パラメーター（ヒゲの比率やクラスタリングの許容値など）が市場のサイクルや時間足によって不適合を起こす場合があります。

**アルゴリズムの調整、パラメーターチューニング、または書き換えなどのコントリビューションを心からお待ちしています：**
- **より優れたクラスタリング**: 1D DBSCANやグリッド密度を用いた、より高精度なサポート・レジスタンスゾーン検出の実装。
- **動的しきい値システム**: ATRを導入し、ピンバーやエンガルフィングのしきい値を自動適応させる。
- **現代的コンセプトの導入**: FVG（Fair Value Gap）やOB（Order Block）などのスマートマニープライスアクションパターンの実装。

アルゴリズムのズレを発見された場合、またはよりエレガントな数学的判定モデルをご存知の場合は、ぜひプルリクエスト等でご参加ください。

## 🔄 CI/CD パイプライン

本プロジェクトは、**GitHub**、**Google Cloud Build**、**Google Cloud Run** を用いた **100% 自動の CI/CD（継続的インテグレーション/デプロイ）**を実装しています。

- **⚡ 自動トリガー (Continuous Integration)**:
  GitHub の `main` 分支にプッシュまたはマージされると、Google Cloud Build の Webhook が自動的に起動し、パイプラインが実行されます。
- **🐳 マルチステージビルド (Dockerized Build)**:
  - `Dockerfile` に基づき、軽量で安全な `node:22-alpine` イメージでビルドされます。
  - **Builderステージ**: 依存関係をインストールし、アセットをバンドル（`npm run build`）。Expressサーバー（TypeScript）を単一のCommonJSモジュール（`dist/server.cjs`）に高速ビルドします。
  - **Runnerステージ**: ビルド済みの静的ファイル（`dist/`）とローカルデータ（`data/`）のみを保持し、開発用ツールや不要なファイルを全て削ることで、非常に小さく起動が速いコンテナを生成します。
- **🚀 シームレスなデプロイ (Continuous Deployment)**:
  コンテナは **Google Artifact Registry** にプッシュされ、数秒で **Google Cloud Run** にローリングアップデートされます。ダウンタイムゼロ、自動スケールを維持します。

### 使用技術 (Tech Stack)

- **フロントエンド:**
  - **React 18** & **TypeScript** — 高い拡張性と型安全性。
  - **Tailwind CSS** — 高コントラストで美しい極客風（ダークスレート・テーマ）のUI設計。
  - **自作SVG描画エンジン** — 肥大化した外部チャートライブラリを排除し、軽量・高性能な純SVGでドラッグ（Pan）や拡大縮小（Zoom）を完全に滑らかに処理。

- **バックエンド:**
  - **Node.js** & **Express** — 複数時間足のローソク足配信およびローカルデータのマージ。
  - **Yahoo Finance API Proxy** — 安全に外部リクエストを中継し、4時間足などのカスタム時間足を動的に生成。
  - **Cron式スケジューラー** — 毎日オフラインでのデータ同期スケジュールを実行。

## 🚀 クイックスタート

### 1. 環境
お使いの環境に **Node.js (v18+)** がインストールされていることを確認してください。

### 2. 依存関係のインストール
プロジェクトのルートディレクトリで、以下を実行します：
```bash
npm install
```

### 3. 開発サーバーの起動
```bash
npm run dev
```
ブラウザで `http://localhost:3000` を開けば、すぐに使用できます。

### 4. プロダクションビルド
```bash
npm run build
```
最適化された静的ファイルとサーバー用のコンパイル成果物が `dist/` に書き出されます。
